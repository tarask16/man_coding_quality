# Этап 2. Рисунок 1.2 — накопление ошибок в ручной процедуре

## Назначение

Этап реализует отдельный генератор линейного графика зависимости вероятности безошибочного выполнения всей процедуры от числа последовательно выполняемых операций.

Используется упрощенная модель независимых операций:

```text
P0(n) = (1 - p)^n,
```

где `p` — вероятность ошибки отдельной операции, `n` — число операций. На графике отображаются четыре значения `p`: 0,005; 0,010; 0,020 и 0,050. Для каждой кривой отмечается первое число операций, при котором `P0(n) <= 0,5`.

PNG формируется с разрешением 300 dpi. SVG сохраняет текстовые элементы и векторные кривые.

## Команда запуска

```powershell
python -m manual_coding_sim.dissertation_figures.figure_1_2_error_accumulation `
  --project-root . `
  --dpi 300 `
  --max-operations 200
```

## Критерии завершения

1. CLI завершается с кодом 0.
2. Созданы PNG и SVG с одинаковым именем основы.
3. Числовые ряды соответствуют формуле `P0(n) = (1 - p)^n`.
4. Для `p = 0,01` первое значение `P0 <= 0,5` достигается при `n = 69`.
5. PNG имеет ширину не менее 3000 пикселей и высоту не менее 1700 пикселей.
6. SVG содержит подписи, формулу, текстовые элементы и векторные кривые.
7. Тесты этапа и полная регрессия проходят успешно.
8. Пользователь визуально подтверждает корректность рисунка.
