# Этап 8. Проверка интервального прогноза качества

## Назначение

Этап выполняет внешнюю проверку интервальной оценки, сформированной в главе 5.
Для каждого сценария проверяется условие:

```text
q_pred_lower <= q_fact <= q_pred_upper
```

Границы интервала, точечный прогноз и параметры неопределенности на этапе 8
не изменяются и не подгоняются по фактическим данным.

## Рассчитываемые показатели

- `coverage_rate`;
- число покрытых сценариев;
- средняя и медианная ширина интервала;
- минимальная и максимальная ширина интервала;
- `miss_lower_count` — факт ниже нижней границы;
- `miss_upper_count` — факт выше верхней границы;
- среднее расстояние до интервала по всем сценариям;
- среднее расстояние среди промахов;
- максимальное расстояние до интервала.

Дополнительно формируются срезы:

- по фактическому классу качества;
- по прогнозному классу качества;
- по доминирующему LDA-фактору;
- по четырем квартилям `uncertainty_score`.

Квартили строятся по рангу неопределенности: `Q1` соответствует наименьшей,
а `Q4` — наибольшей неопределенности.

## Команда запуска

```powershell
python -m manual_coding_sim.validation.chapter6_runner `
  --project-root . `
  --config configs\chapter6.yaml `
  --validate-interval-prediction
```

## Тесты этапа

```powershell
pytest tests\validation\test_interval_prediction_validator.py -q
pytest tests\validation -q
pytest -q
```

После полной регрессии необходимо повторить CLI этапа 8.
