# Отчет этапа 11. Формирование итогового отчета главы 5

## Назначение этапа

Этап 11 завершает отчетный слой программного блока главы 5. Реализовано формирование итогового JSON- и Markdown-отчета по результатам априорного прогнозирования качества ручного кодирования.

## Реализованные изменения

- Реализован `Chapter5ReportBuilder`.
- Добавлен CLI-флаг `--build-report`.
- Итоговый отчет формируется в двух форматах:
  - `reports/chapter5/chapter5_prediction_report.json`;
  - `reports/chapter5/chapter5_prediction_report.md`.
- В отчет включены:
  - сводка методической безопасности;
  - результаты нормировки априорных признаков;
  - сводка латентной компоненты качества;
  - частные прогнозные критерии;
  - интегральный показатель `Q_pred`;
  - интервальная неопределенность;
  - распределение сценариев по прогнозным классам качества.

## Методическая безопасность

Итоговый отчет строится только по артефактам главы 5, полученным в априорном контуре. Фактические критерии качества, целевые метки и диагностические признаки в отчет не загружаются.

## Контрольный запуск

Команда:

```powershell
python -m manual_coding_sim.prediction.chapter5_runner `
  --project-root . `
  --config configs/chapter5.yaml `
  --run-full-pipeline `
  --build-report
```

Контрольный результат:

- строк итогового отчета: 150;
- `Q_pred` min: 0.10344477176221134;
- `Q_pred` max: 0.8893015161715213;
- прогнозные классы качества: low = 67, medium = 49, high = 34;
- проверка методической утечки: пройдена;
- полный CLI-контур: выполнен.
