# Этап 4. Защита от утечки фактических признаков

## Статус

Этап 4 реализован и закрыт.

## Реализованные изменения

Добавлен расширенный контроль методической утечки в модуле:

```text
src/manual_coding_sim/prediction/chapter5_leakage_guard.py
```

Реализованы возможности:

- проверка запрещенных колонок по шаблонам `fact_*`, `actual_*`, `target_*`, `targets_*`, `*_fact`, `*_actual`, `*_target`;
- блокировка целевых показателей `q_acc`, `q_time`, `q_effort`, `q_res`, `q_rep`, `q_fit`, `integral_quality`;
- блокировка распространенных фактических колонок `fact_duration_sec`, `fact_error_count`, `fact_reject_count`, `fact_success`;
- проверка без учета регистра;
- формирование структурированного результата `Chapter5LeakageCheckResult`;
- сохранение JSON-отчета проверки утечки;
- русскоязычные сообщения ошибок и диагностические сообщения CLI.

## Интеграция

Проверка утечки встроена в загрузку `prior_features.csv`:

```text
src/manual_coding_sim/prediction/chapter5_data_loader.py
```

Теперь загрузчик главы 5 прерывает выполнение, если во вход априорного прогноза попали фактические или целевые признаки.

Также проверка подключена к CLI:

```text
src/manual_coding_sim/prediction/chapter5_runner.py
```

При запуске с флагом `--validate-inputs` формируется отчет:

```text
reports/chapter5/chapter5_leakage_report.json
```

## Проверка на фактических данных

Команда:

```powershell
$env:PYTHONPATH = "src"
python -m manual_coding_sim.prediction.chapter5_runner `
  --project-root . `
  --config configs\chapter5.yaml `
  --validate-inputs
```

Результат:

```text
Данные главы 5 успешно загружены: 150 сценариев, 3 латентных фактора.
Ключи объединения: ('scenario_id', 'protocol_id')
Проверка методической утечки: пройдена.
Отчет проверки утечки сохранен: reports/chapter5/chapter5_leakage_report.json
```
