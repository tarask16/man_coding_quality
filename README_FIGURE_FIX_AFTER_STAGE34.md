# Исправление генератора рисунка 6.7 после этапа 34

## Причина

Фактический `reports/chapter6/baseline_comparison.csv` использует идентификатор
`chapter5_model` для полной модели главы 5. Генератор рисунка 6.7 не включал
этот идентификатор в таблицу допустимых алиасов и завершался исключением.

## Изменение

В `_MODEL_ALIASES` добавлено соответствие:

```python
"chapter5_model": "full"
```

Тест нормализации имён обновлён: теперь он непосредственно проверяет
идентификатор `chapter5_model`.

## Методические последствия

Численные значения, порядок моделей, подписи, интерпретация рисунка и выходные
пути не изменены. Исправление затрагивает только совместимость генератора с
фактической схемой baseline-отчёта.

Тесты в рабочем окружении не запускались. PNG и SVG повторно сформированы с
использованием идентификатора `chapter5_model`.
