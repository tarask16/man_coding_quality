# Синтетические контроли основной гипотезы

## Назначение

Пакет содержит два полностью синтетических датасета, совместимых со схемой
`reports/chapter6/validation_dataset.csv` после этапа 3 главы 6.

**Критическое ограничение:** положительный контроль специально сгенерирован так,
что латентный профиль входит в механизм формирования фактического качества.
Поэтому его результат нельзя представлять как эмпирическое доказательство
основной гипотезы диссертации. Он подтверждает только следующее:

> программный и статистический контур способен обнаружить известный заранее
> вклад латентной компоненты, когда такой вклад действительно присутствует.

Для доказательства основной гипотезы по существу требуется независимый реальный
или хотя бы отдельно спроектированный имитационный эксперимент, параметры
которого не подбирались по желаемому результату.

## Состав

- `data/positive_control/validation_dataset.csv` — 600 синтетических сценариев;
- `data/positive_control/validation_dataset_test.csv` — независимая тестовая часть, 120 сценариев;
- `data/null_control/validation_dataset.csv` — 600 сценариев без вклада LDA в целевую величину;
- `data/null_control/validation_dataset_test.csv` — нулевая тестовая часть, 120 сценариев;
- `split_assignments.csv` — разбиение `development/calibration/test`;
- `scripts/generate_synthetic_controls.py` — воспроизводимый генератор;
- `scripts/run_synthetic_control.ps1` — изолированный запуск этапов 9–10;
- `reports/...` — контрольные результаты штатного программного контура;
- `metadata/generation_parameters.json` — параметры генерации;
- `metadata/expected_results.json` — ожидаемые итоговые показатели.

## Механизм положительного контроля

Для каждого частного критерия фактическое качество формируется как

```text
Q_fact,j = 0.55 * F_j + 0.45 * L_j + epsilon_j,
```

где:

- `F_j` — априорная feature-компонента;
- `L_j` — латентная компонента, связанная с профилем `theta`;
- `epsilon_j` — малый случайный шум результата.

Прогноз главы 5 использует обе компоненты в тех же фиксированных долях. Поэтому
положительный контроль является **positive control**: при корректной реализации
полная модель должна превосходить `prior_only_baseline` и `theta_only_baseline`.

## Механизм нулевого контроля

В нулевом контроле:

```text
Q_fact,j = F_j + epsilon_j.
```

Латентная компонента не участвует в истинном механизме качества. Поэтому
`prior_only_baseline` должен превосходить полную модель. Этот контроль нужен,
чтобы показать, что программный контур не подтверждает LDA автоматически.

## Результаты на независимой тестовой части положительного контроля

| Модель | MAE | RMSE | Spearman | Kendall | Accuracy | Balanced Accuracy | Macro F1 |
|---|---:|---:|---:|---:|---:|---:|---:|
| Mean baseline | 0.195962 | 0.228533 | -0.297181 | -0.213941 | 0.333333 | 0.333333 | 0.166667 |
| Prior-only | 0.096374 | 0.119425 | 0.915529 | 0.749300 | 0.741667 | 0.742340 | 0.738467 |
| Theta-only | 0.117550 | 0.144078 | 0.787819 | 0.564426 | 0.716667 | 0.717761 | 0.715048 |
| Полная модель | **0.020222** | **0.025300** | **0.988867** | **0.913725** | **1.000000** | **1.000000** | **1.000000** |

Парный bootstrap, 1000 повторов:

```text
устойчивых преимуществ полной модели: 18 из 18;
устойчивых преимуществ baseline:      0;
неопределенных сравнений:             0.
```

Относительно `prior_only_baseline` доверительные интервалы всех шести разностей
не включают ноль. То есть положительный контроль подтверждает заложенную в него
гипотезу одновременно по MAE, RMSE, Spearman, Kendall, Accuracy и Macro F1.

## Результаты нулевого контроля

На тестовой части нулевого контроля `prior_only_baseline` имеет:

```text
MAE       = 0.024071
Spearman  = 0.990347
Macro F1  = 0.963123
```

Полная модель имеет:

```text
MAE       = 0.100217
Spearman  = 0.907105
Macro F1  = 0.815599
```

Во всех шести парных сравнениях с `prior_only_baseline` bootstrap устойчиво
поддерживает baseline. Это корректное поведение при отсутствии истинного вклада
латентного профиля.

## Безопасный запуск в отдельном каталоге

Из корня проекта:

```powershell
.\.venv\Scripts\Activate.ps1

.\synthetic_controls\scripts
un_synthetic_control.ps1 `
  -ProjectRoot . `
  -Control positive
```

Для нулевого контроля:

```powershell
.\synthetic_controls\scripts
un_synthetic_control.ps1 `
  -ProjectRoot . `
  -Control null
```

Скрипт создает изолированный каталог:

```text
synthetic_runs/positive
synthetic_runs/null
```

Рабочие артефакты глав 3–6 в основном проекте не заменяются.

## Допустимая формулировка в диссертации

Корректно:

> На синтетическом положительном контроле, в котором независимый вклад
> латентного профиля был заранее включен в механизм формирования фактического
> качества, программный контур устойчиво обнаружил преимущество полной модели
> по всем заданным метрикам. На нулевом контроле преимущество LDA не было
> обнаружено относительно prior-only модели. Это подтверждает чувствительность
> и специфичность разработанной процедуры проверки гипотезы.

Некорректно:

> Синтетический датасет доказал практическое превосходство метода LDA.

Практическое доказательство требует независимых фактических данных.
