# Контракт адаптера к базовому пакету

## Назначение

Расширение `manual_coding_sim_decoding` не изменяет и не подменяет пакет
`manual_coding_sim`. Связь выполняется через явно перечисленные публичные
классы и структуры данных.

## Разрешенные зависимости этапа 1

- `manual_coding_sim.types`;
- `manual_coding_sim.message_model`;
- `manual_coding_sim.procedure_model`;
- `manual_coding_sim.operator_model`;
- `manual_coding_sim.condition_model`;
- `manual_coding_sim.error_model`;
- `manual_coding_sim.control_model`;
- `manual_coding_sim.protocol_simulator`.

Точный перечень символов хранится в
`configs/decoding_stage1.yaml`. Расширение не выполняет monkey patch,
наследование с изменением состояния базовых экземпляров или запись в
`src/manual_coding_sim`.

## Инварианты совместимости

1. Имя нового Python-пакета не пересекается с именем базового пакета.
2. Все выходные каталоги находятся в `extensions/decoding_simulation/`.
3. Базовые файлы сопоставляются с SHA-256-манифестом этапа 0.
4. Несовпадение хэша, отсутствие модуля или публичного символа приводит к
   отрицательному статусу контракта и коду возврата CLI `2`.
5. На этапе 1 адаптер только проверяет интерфейсы и ничего не моделирует.

## Граница последующих этапов

Материальные типы `EncodedMessage`, обратная процедура и сквозные метрики
добавляются только в пакет `manual_coding_sim_decoding`. Перенос этих типов в
базовый пакет возможен лишь после отдельного решения по итогам этапа 12.
