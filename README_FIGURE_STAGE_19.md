# Этап 19. Рисунок 4.5 — наиболее весомые токены латентных факторов

## Назначение

Этап формирует три горизонтальные диаграммы наиболее весомых токенов тем модели `LDA_prior`. Все панели используют единую шкалу веса, что позволяет сопоставлять не только порядок токенов внутри темы, но и абсолютный уровень их вероятностной массы.

## Входные данные

- `reports/chapter4/topic_word.csv` — веса всех токенов для тем 0–2;
- `reports/chapter4/topic_interpretation.json` — доминирующие токены и интерпретационные метаданные.

Ожидаемая структура `topic_word.csv`:

```text
topic_id, token_id, token, document_frequency, weight
```

Для фактической модели используются:

- 3 темы;
- 96 токенов в словаре каждой темы;
- 288 строк отчёта;
- нормировка весов каждой темы до единицы.

## Содержание рисунка

Показаны первые 10 токенов для каждого фактора:

1. тема 0 — процедурная трудоёмкость;
2. тема 1 — операционный риск;
3. тема 2 — благоприятные условия.

Технические имена токенов преобразуются в краткие русские подписи. Для каждой панели дополнительно выводятся сумма весов top-10 и доминирующий признак.

## Методическое ограничение

Высокий вес означает характерность токена для латентной темы. Он не является коэффициентом причинного влияния на фактическое качество и не заменяет внешнюю проверку интерпретации факторов.
