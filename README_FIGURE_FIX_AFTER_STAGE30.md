# Исправление после этапа 30

## Причина

Тест `test_absolute_errors_require_at_least_three_points` ожидает, что
`merge_residual_points()` выполнит только объединение и структурную проверку двух
согласованных наблюдений, а ограничение `не менее трёх` будет проверено функцией
`calculate_absolute_errors()`.

Предыдущая реализация вызывала полную проверку `validate_points()` уже внутри
`merge_residual_points()`, поэтому исключение возникало раньше ожидаемой точки.

## Изменение

В `figure_6_2_residuals_vs_q_fact.py` добавлена функция
`validate_merged_points()`, которая проверяет:

- уникальность составных ключей;
- конечность значений;
- принадлежность `q_pred` и `q_fact` диапазону `[0; 1]`.

`merge_residual_points()` вызывает только эту структурную проверку.
Требование минимум трёх сценариев и непостоянства `Q_fact` остаётся в
`validate_points()` и применяется при расчёте сводки рисунка 6.2.
Требование минимум трёх наблюдений для рисунка 6.3 остаётся в
`calculate_absolute_errors()`.

## Результат

Рисунки 6.2 и 6.3 повторно сформированы в PNG и SVG. Их численные данные и
визуальная интерпретация не изменились.

Тесты в рабочем окружении не запускались.
